Design a complete mobile app UI for "Bubbles" — an AI-powered real-time
communication assistant for iOS and Android. The app listens to live
conversations, provides real-time coaching suggestions, builds a personal
knowledge graph of people and relationships, and lets users query their
conversation history via a ChatGPT-style consultant chat.

Generate every screen, overlay, dialog, bottom sheet, and empty state.
Both light and dark mode variants are required for every screen.

────────────────────────────────────────────────────────────────────────────────
DESIGN SYSTEM  (apply to every screen — non-negotiable)
────────────────────────────────────────────────────────────────────────────────

App name:        Bubbles
Tagline:         Your AI Conversation Wingman

Font family:     Manrope (Google Fonts)
  - Display / hero text:    Manrope 800 ExtraBold
  - Section headers:        Manrope 700 Bold
  - Body / labels:          Manrope 500 Medium
  - Captions / hints:       Manrope 400 Regular

Color palette:
  Primary blue          #13A4EC
  Primary dark          #0B8BC9
  Primary light         #6ECBF5

  Background dark       #101C22
  Background light      #F6F7F8

  Surface dark          #192B33
  Surface dark highlight #233C48
  Surface light         #FFFFFF

  Chat bubble dark      #1C2A33
  Chat bubble user      rgba(19, 164, 236, 0.10)

  Text primary dark     #FFFFFF
  Text secondary        #94A3B8  (slate-400)
  Text muted            #64748B  (slate-500)

  Success green         #22C55E
  Error red             #EF4444
  Warning amber         #F59E0B

  Entity type colors:
    person              #818CF8  (indigo)
    place               #34D399  (emerald)
    organization        #FBBF24  (amber)
    event               #F87171  (red)
    object              #60A5FA  (blue)
    concept             #A78BFA  (violet)

Corner radius scale:
  sm = 8px   md = 12px   lg = 16px   xl = 20px   xxl = 24px   pill = 9999px

Spacing scale:
  xs = 6px   sm = 12px   md = 18px   lg = 24px   xl = 32px

Shadow style: soft, layered — no harsh drop shadows
Glassmorphism: used on overlays and modal panels
  - backdrop-filter: blur 20px
  - background: surface color at 80% opacity
  - border: 1px solid white at 10% opacity

Status bar: always transparent, icons adapt to theme
Navigation: no bottom navigation bar — navigation happens via gesture,
            buttons within screens, and a side drawer (consultant screen only)
Platform: mobile-first, safe area aware, no notch overlap

────────────────────────────────────────────────────────────────────────────────
ADDITIONAL DESIGN NOTES
────────────────────────────────────────────────────────────────────────────────

1. ALL interactive elements must have a clear press/hover state:
   - Cards: slight scale-down (0.97) + brightness darken on tap
   - Buttons: opacity 0.85 while pressed

2. ALL text inputs follow the same pattern:
   - Rounded corners 16px (lg)
   - 1px outline border, surface dark highlight color normally
   - On focus: 2px primary blue outline, no fill change
   - Prefix icon in primary blue
   - Error state: 2px error red border + red error text below field

3. Color-coding is consistent throughout:
   - Primary blue = Wingman features, primary calls to action
   - Purple (#818CF8) = Consultant AI features
   - Green = success, connected, completed
   - Red = error, disconnected, destructive action
   - Amber = warning, pending

4. Both light and dark mode must be fully designed. In dark mode:
   - NO pure black (#000) backgrounds — always #101C22
   - Surface cards use #192B33, not dark grey
   - Text is NOT pure white — use #F8FAFC (near white) for primary text

5. The "Bubbles" logo:
   - Icon: speech bubble outline with three dots inside + a small sparkle/star
     in the top-right corner of the bubble. Clean, modern, single-color.
   - Works on both dark (#101C22) and light (#F6F7F8) backgrounds.
   - Primary blue version for light mode. White version for dark mode.
