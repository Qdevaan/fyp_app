     Files: server/.env.example

SECTION 2: CODE QUALITY — FLUTTER
-----------------------------------------------------------

2.1  Extracted reusable ChatBubble widget
     Files: lib/widgets/chat_bubble.dart (NEW)
            lib/screens/new_session_screen.dart
            lib/screens/consultant_screen.dart
            lib/screens/sessions_screen.dart
     - Eliminated ~40 lines of duplicated chat bubble UI code
     - Supports speaker labels, custom content widgets (Markdown), theme-aware colors

2.2  Cleaned up main.dart
     Files: lib/main.dart
     - Removed all commented-out imports and dead route definitions
     - Added global error handler via FlutterError.onError

2.3  Replaced print() with debugPrint() everywhere
     Files: All .dart files in lib/services/ and lib/screens/
     - print() can overflow in release mode; debugPrint() is production-safe
     - Fixed in: api_service, connection_service, deepgram_service, livekit_service,
                 new_session_screen

2.4  Fixed deprecated WidgetsBinding.instance.window
     Files: lib/screens/consultant_screen.dart
     - Replaced with View.of(context).viewInsets.bottom (modern Flutter API)

2.5  Fixed dark mode text color bug
     Files: lib/screens/new_session_screen.dart
     - Changed Colors.black87 to use theme's foreground color (fg parameter)
     - Changed hardcoded Colors.grey to theme.colorScheme.onSurfaceVariant

2.6  Fixed LiveKitService Provider rebuild issue
     Files: lib/main.dart, lib/services/livekit_service.dart
     - Changed from ProxyProvider (creates new instance every update) to
       ChangeNotifierProxyProvider with updateApiService() method
     - Preserves LiveKit connection state across Provider rebuilds

2.7  Consistent navigation
     Files: lib/screens/settings_screen.dart
     - Replaced 2 instances of MaterialPageRoute push with Navigator.pushNamed
     - Removed unused ConnectionsScreen import
